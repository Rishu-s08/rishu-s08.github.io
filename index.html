SmartCart — Details

1. Core Purpose
Cartelle is designed to simplify shopping and reminders by combining:
• Smart shopping lists
• Location-based reminders
• Time-based reminders
• Clean, elegant, modern UI

2. Features & Details
A. Smart Shopping Lists
• Create and manage multiple shopping lists.
• Add items with:
   – Name
   – Quantity
   – Category (e.g., grocery, electronics)
   – Optional notes
• Mark items as completed.
• Planned enhancement: Predictive suggestions based on previous items.
How we make it:
   – UI: Flutter with Riverpod for state management.
   – Data: Local DB (Hive/SQLite) + sync with backend.
   – Logic: Lists stored per user, each item having a status (pending/completed).

B. Reminders
• Time-based reminders (like alarms for tasks).
• Location-based reminders:
   – Trigger when entering/leaving a certain radius from a saved location.
   – Perfect for "Buy milk when near XYZ store" use case.
• GPS toggle: Ability to turn tracking on/off to save battery.
How we make it:
   – Time-based: Flutter awesome_notifications or similar.
   – Location-based: flutter_background_geolocation for geofencing.
   – Backend: Save reminder details for sync.

C. Location Management
• Save frequently visited places (shops, markets, etc.).
• Each location can have:
   – Name
   – Latitude/Longitude
   – Radius (for geofence trigger)
• Option to edit or delete locations.
• Check if a shopping list is using a location before deletion.
How we make it:
   – UI: Google Maps Flutter package (marker + draggable).
   – Data: Location table linked with lists/reminders.

D. Add Screen (Quick Create)
• Single, central "Add" tab to:
   – Create a new list.
   – Add items to a list.
   – Create a reminder.
   – Save a location.
• Designed for speed — minimal taps to add something.
How we make it:
   – Navigation: go_router for deep linking.
   – UI: Floating menu / segmented add form.

E. Profile & Analytics
• Track spending trends per category/list.
• See monthly breakdown.
• Option to export data.
• Planned: AI-based spending insights.
How we make it:
   – Charts: fl_chart for graphs.
   – Backend: Store completed items with cost metadata.

F. Notifications
• Get notified when:
   – You’re near a location with a pending task.
   – A reminder is due.
• In-app notification history for missed alerts.

G. User Authentication
• Firebase Email/Password login.
• Email verification.
• Google Sign-In (future).
• Secure user data sync.

H. Design
• Soft, elegant UI inspired by the cupcake theme.
• Rounded corners, pastel colors, minimal shadows.
• Consistent typography.
• Smooth transitions & animations.

3. Tech Stack
• Frontend: Flutter (Riverpod, go_router)
• Backend: Node.js + Express (planned)
• Database: MongoDB (remote), Hive (local caching)
• Maps & Location: Google Maps API + flutter_background_geolocation
• Auth: Firebase Authentication
• Notifications: awesome_notifications
• Hosting: VPS/Docker (future)

Faculty Feedback System — Details

📌 Core Purpose
- Collect structured feedback from students about faculty performance.
- Maintain teaching quality and identify areas for improvement.
- Simplify the feedback process with a modern, intuitive interface.

🎯 Key Features for Students
- Secure Login – Role-based access (students, admins).
- Relevant Forms Only – Students see only their semester & branch forms.
- No Repeat Submissions – Forms disappear after submission.
- Static Questions – Pre-defined on teaching skills, punctuality, clarity, etc.
- Multiple Response Types – Ratings (stars/scales) + optional comments.
- Deadline Alerts – Push/email notifications for pending feedback.
- Mobile-Friendly UI – Clean, fast, and easy to use.

🛠 Features for Administrators
- Faculty Management – Add, edit, or remove faculty profiles.
- Feedback Form Creation – Pre-set or custom questions with scheduling.
- Role-Based Access – Admins control creation, viewing, and management rights.
- Detailed Analytics – Averages, trends, question-wise performance.
- Data Export – Generate PDF/CSV reports for reviews or audits.
- Integration Ready – Connect with institutional SSO or student portals.

📊 Reporting & Insights
- Performance Reports – Individual & department-wide summaries.
- Trend Analysis – Track improvement over time.
- Strengths & Weaknesses – Highlight excellence and areas needing growth.
- Accreditation Support – Export-ready data for documentation.

🔒 Security & Authenticity
- Prevents multiple submissions per student per form.
- Secure storage and encrypted data transfer.
- Audit logs for admin actions.

🌟 Additional Options
- Offline mode with sync when online.
- Multi-language support for diverse campuses.
- Customizable themes to match institution branding.

Backend

Features: API gateway, auth service, post service, docker-compose setup, Redis caching.

Blog App

Developed using Flutter with Bloc state management and Supabase backend, this app offers a robust blogging platform with the following features:

Core Features:
• Full CRUD operations: Users can create, read, update, and delete blog posts seamlessly.
• Rich Text Editor: Provides formatting options like bold, italics, lists, links, and more for creating engaging content.
• Image Upload: Users can upload images to accompany their posts, enhancing visual appeal.
• Categories & Tags: Organize posts by categories for easy browsing and filtering.
• User Authentication: Secure signup, login, and session management powered by Supabase Auth.
• Responsive Design: Optimized UI for both mobile and web platforms to ensure smooth user experience.

Technical Highlights:
• State Management: Bloc pattern ensures scalable and maintainable state handling across the app.
• Backend Integration: Supabase handles real-time database, authentication, and file storage.
• API Communication: Efficiently manages network requests with error handling and loading states.
• Data Validation: Ensures input integrity before saving posts or updating user profiles.
• Performance Optimization: Lazy loading of posts and images to improve speed and reduce resource usage.

User Experience:
• Intuitive UI with clean layouts and smooth transitions.
• Real-time updates: Posts and comments reflect instantly with Supabase’s real-time capabilities.
• Accessibility considerations for wider user reach.

This app demonstrates comprehensive full-stack development skills, combining Flutter frontend expertise with modern backend technologies and clean architecture principles.

Reddit Clone

Reddit Clone
Responsive full-stack Reddit clone app built for Android, iOS, and Web platforms using Flutter.

Features:
• Google & Guest Authentication for easy access.
• Create and join communities to connect with others.
• Community profiles with avatar, banner, and member counts.
• Edit community description and avatar.
• Post creation supporting link-only, photo, and text-only posts.
• Display posts from communities the user is part of.
• Upvote and downvote system to rank posts.
• Commenting on posts to engage in discussions.
• Award posts with custom awards.
• Karma system that updates based on user activity.
• Add moderators with permissions to manage communities.
• Moderators can remove or delete posts.
• User profiles with customizable avatar and banner.
• Theme switch for light and dark modes.
• Cross-platform support with a responsive UI.
• Latest posts feed available to guest users.

Check out the full tutorial series on YouTube by Rivaan Ranawat to learn how to build this app step-by-step.
